# Context Graphs Experiment

Research experiment exploring context graphs as competitive infrastructure in the age of AI.

## Repo Type
type: research

## Quick Context

This is a **research experiment** exploring the strategic and technical dimensions of context graphs. Key thesis: context graphs are **projections** of datagraphs, not standalone assets.

## Core Framework

```
┌─────────────────────────────────────────────────────────────┐
│  KNOWLEDGE GRAPHS  │  Semantic layer - what things mean     │
├────────────────────┼────────────────────────────────────────┤
│     DATAGRAPHS     │  Learning layer - behavioral data      │
│                    │  with network effects (strategic)      │
├────────────────────┼────────────────────────────────────────┤
│   CONTEXT GRAPHS   │  Action layer - moment-specific        │

└────────────────────┴────────────────────────────────────────┘
```

## Key Sources

| Venkatraman/Govindarajan (HBR 2022) | Datagraphs create data network effects through product-in-use data |

## Research Questions

1. **Architectural**: How do you represent the three layers technically?
2. **Strategic**: What defines the "projection function" from datagraph → context?
3. **Practical**: What does a decision trace actually look like?
4. **Competitive**: Can you build value without owning the underlying datagraph?
